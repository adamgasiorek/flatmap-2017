1. OFERTY
1.1 GET MARKERS: http://localhost:9999/marker/get?params

DESCRIPTION:

wyszukanie znaczników znajdujących się w odpowiedniej lokalizacji tj:
szerokość geograficzna - latitude [x1,x2]
długość geograficzna - longitude [y1,y2]
Oraz poddanych odpowiedniemu filtrowaniu

PARAMS:

  • Wymagane:

  • argumenty x1, x2, y1, y2 SĄ WYMAGANE, inaczej response: 400 – BAD REQUEST
  • argumenty w dowolnej kolejności
  • argumenty to realne współrzędne z zakresu:
    • x1,x2 - szerokość geograficzna - latitude [-90,90] - przedział domknięty
    • y1,y2 - długość geograficzna - longitude [-180,180] - przedział domknięty
  • jeżeli podamy argumenty spoza zakresu: response 400 - bad request
  • jeżeli x1>x2 albo y1 < y2 response code = 400 - bad request -! nie mylić z y1>y2 co JEST dopuszczalne i normalne
  • przedziały zamknięte czyli jak znacznik o współrzędnych np. x1,y1 to ZOSTANIE uwzględniony

  • Argumenty filtrowania:

    Wielkość liter w nazwie argumentu bez znaczenia
    Niepoprawna nazwa argumentu - 400 Bad Request
    Niepoprawna wartość argumentu - 400 - Bad Request

    • offerType = (int z zakresu OfferType) - dopuszczalne kilka wartości oddzielonych - np: 1-2 znaczy typ 1 lub typ 2
    • propertyTypeEnum = (int z zakresu PropertyType) - dopuszczalne kilka wartości oddzielonych - np: 1-2 znaczy typ 1 lub typ 2
    • price = minimalna-maksymalna-waluta (int z zakresu CurrencyType) np 400-600-0 tzn od 400 do 600 zł albo sama minimalna 400--0 czyli 400 zł
    • lastAdded - int z zakresu LastAddedType - jedna warosc
    • buildingType - wieloktornego wyboru - dowlony int ale skuteczne beda tylko z zakresu ENUM_BUILDING_TYPE
    • personcount = wielokrotnego wyboru np 2-3-4 czyli 2 3 albo 4 osoby
    • heatingtype = (int z zakresu HeatingType) - dopuszczalne kilka wartości oddzielonych - np: 1-2 znaczy typ 1 lub typ 2
    • area = zakres - min-max np 40-70 ALBO sama powierzchnia minimalna np 40- ALBO sama maksymalna wtedy -70
    • roomcount = wielokrotnego wyboru np 2-3-4 czyli 2 3 albo 4 pokoje
    • floor = wielokrotnego wyboru np 2-3-4 czyli 2 3 albo 4 pietro
    • year - zakres min-max np 1995- czyli nowsze niz 1995 albo 1970-1980 czyli zakres
    • furnished = - true/false
    • balcony - true/false
    • lift - true/false
    • basement - true/false
    • parking - true/false
    • pets - true/false
    • garden - true/false
    • clima - true/false
    • smoking - true/false
    • isFromAgency - true/false

RETURN:

  • 200 OK : json lista markerów o polach:
    • id
    • offerType:
      1. HIRE
      2. SELL
      3. OTHER
    • propertyTypeEnum:
      1. BLOCK
      2. TENEMENT
      3. DETACHED_HOUSE
      4. TERRACED_HOUSE
      5. LOFT
      6. OTHER
    • latitude
    • longitude
  • 400 BAD REQUEST: niepoprawne argumenty

EXAMPLE:

get:

http://localhost:9999/marker/get?x1=50.0&x2=60&y1=20&y2=21

http://localhost:9999/marker/get?x1=50.1&x2=50.2&y1=20&y2=22&offertype=1&propertytype=1

http://localhost:9999/marker/get?x1=3&y1=100&x2=90&y2=1&offertype=1&propertytype=1-4

repsonse:

[{"id":1101,"offerType":0,"propertyTypeEnum":0,"latitude":50.126440142856325,"longitude":20.04263720643732},{"id":1106,"offerType":2,"propertyTypeEnum":5,"latitude":50.16622178264657,"longitude":20.037098474390632},{"id":1121,"offerType":2,"propertyTypeEnum":2,"latitude":50.10687477723991,"longitude":20.019202017140373}]
